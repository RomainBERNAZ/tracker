## 2.1 Replayer — Step-by-Step Hand Visualization

### Goals
1. Load any hand from DB and display its progression
2. Navigate forward/backward through action timeline
3. Display board, stacks, and actions at each street
4. Show showdown (if applicable)
5. Visualize chip flow and pot growth

### Data Model

**ReplayerState** (from hand_actions + hand_players):
```rust
pub struct ReplayerState {
    pub hand_id: String,
    pub table_name: String,
    pub timestamp: DateTime<Utc>,
    pub players: Vec<ReplayerPlayer>,          // Position → Player info
    pub button_pos: usize,
    pub board: Vec<Card>,                      // Progressive: [], [f1,f2,f3], [f1,f2,f3,t], [f1,f2,f3,t,r]
    pub current_step: usize,                   // 0..total_actions
    pub total_steps: usize,
    pub steps: Vec<ReplayerStep>,              // Indexed timeline
}

pub struct ReplayerPlayer {
    pub position: usize,
    pub name: String,
    pub starting_stack: i64,
    pub current_stack: i64,           // At current_step
    pub hole_cards: Option<[Card; 2]>,
    pub folded: bool,
}

pub struct ReplayerStep {
    pub step_number: usize,
    pub street: Street,                // Preflop, Flop, Turn, River
    pub actor_pos: usize,
    pub action: ActionType,            // Bet, Raise, Call, Check, Fold, AllIn
    pub amount: i64,
    pub pot_size_after: i64,
    pub player_stacks_after: Vec<i64>, // Updated stacks for all players
    pub description: String,           // "PlayerA bets 50", etc.
}
```

### Queries to Implement (Rust / session_read_model)

**1. Load Hand for Replay** (new query)
```rust
pub fn load_hand_for_replay(db: &Connection, hand_id: &str) -> Result<ReplayerState>
```
- Fetch hand + players + actions + showdown
- Build initial ReplayerState at step 0
- Calculate stacks at each step by replaying actions

**2. Get Showdown Cards** (new)
```rust
pub fn get_showdown_cards(db: &Connection, hand_id: &str) -> Result<Vec<(usize, [Card; 2])>>
```
- Returns positions + hole_cards from hand_showdown

### Backend Tasks

- [ ] Implement `load_hand_for_replay()` query (replay action timeline, calculate stacks)
- [ ] Implement `get_showdown_cards()` query
- [ ] Create IPC command: `replay_hand(hand_id: String) → ReplayerState`
- [ ] Add DB index: `hand_actions.hand_id` (already exists)
- [ ] Unit tests: Replayer state calculation for 5 test hands

### Frontend Tasks

- [ ] Create `Replayer.tsx` component
  - State: `currentStep`, `replayerState`, `loaded`
  - Display: Board, players, pot, stacks
  - Controls: First / Previous / Next / Last buttons, step slider

### Acceptance Criteria

## 2.2 Filters — Search & Filter Hands

### Goals
1. Filter hands by position (SB, BB, BTN)
2. Filter by action (fold, all-in, showdown, etc.)
3. Filter by result (won, lost, break-even)
4. Filter by stake/blind level
5. Stack filter (starting, ending)
6. Combine multiple filters (AND logic)

### Data Model

**FilterCriteria** (frontend state):
```rust
pub struct FilterCriteria {
    pub positions: Vec<usize>,        // Empty = all; [0, 1, 2] = specific
    pub actions: Vec<ActionType>,     // Fold, AllIn, Showdown, etc.
    pub results: Vec<ResultType>,     // Won, Lost, BreakEven
    pub blind_levels: Option<(f64, f64)>, // Min/max BB
    pub stack_range: Option<(i64, i64)>,  // Min/max starting stack
}

pub enum ResultType {
    Won,
    Lost,
    BreakEven,
}
```

### Queries to Implement (Rust / session_read_model)

**Search Hands by Criteria** (new)
```rust
pub fn search_hands(
    db: &Connection,
    player_name: &str,
    criteria: &FilterCriteria,
    limit: usize,
    offset: usize,
) -> Result<(Vec<HandSummary>, usize)>  // (results, total_count)
```
- Join hands + hand_players + ledgers
- Build WHERE clauses dynamically
- Return paginated results + total count

### Backend Tasks

- [ ] Add `FilterCriteria` type to `session_read_model`
- [ ] Implement `search_hands()` query (complex JOIN + WHERE building)
- [ ] Create IPC command: `search_hands(criteria) → (results, count)`
- [ ] DB indexes: 
  - `ledgers.player_pos, ledgers.realized_cev` (for result filtering)
  - `hand_players.player_name` (already exists)
- [ ] Unit tests: Filter queries with 5+ criteria combinations

### Frontend Tasks

- [ ] Create `FilterPanel.tsx` component
  - Checkboxes: Positions (SB, BB, BTN)
  - Multi-select: Actions (Fold, All-In, Showdown, etc.)
  - Radio buttons: Results (Won, Lost, Break-Even, All)
  - Sliders: Blind level range, stack range
- [ ] State management: FilterCriteria in Zustand store
- [ ] Create `HandList.tsx` component (paginated, sortable by date/cEV/result)
- [ ] Connect filters to hand list (auto-update on filter change)
- [ ] Add "Clear Filters" / "Save Filter as Preset" (optional)
- [ ] Unit tests: Filter updates, pagination

### Acceptance Criteria

- ✓ Can filter by position + action + result
- ✓ Filters combine correctly (AND logic)
- ✓ Hand list updates immediately on filter change
- ✓ Pagination works (50 hands/page)
- ✓ Performance: Filter query ≤ 200ms on 900 hands
- ✓ All unit tests passing

**Estimated Duration**: 5–7 days (backend 2d + frontend 3d + testing 1-2d)

## 2.3 Session Summary — Tournament Overview Dashboard

### Goals
1. Display session date, time range, hand count
2. Total realized cEV & breakdown by position
3. Stats: win rate, avg hand played, longest session
4. Charts: cEV per hour, win/loss distribution
5. Quick access to best/worst hands

### Data Model

**SessionSummary**:
```rust
pub struct SessionSummary {
    pub session_date: Date,
    pub hand_count: usize,
    pub session_duration: Duration,
    pub session_start: DateTime<Utc>,
    pub session_end: DateTime<Utc>,
    pub total_cev: i64,
    pub avg_cev_per_hand: f64,
    pub by_position: HashMap<usize, PositionStats>, // SB, BB, BTN
    pub stats: SessionStats,
}

pub struct PositionStats {
    pub hands_played: usize,
    pub total_cev: i64,
    pub avg_cev: f64,
    pub win_rate: f64,
}

pub struct SessionStats {
    pub hourly_cev: Vec<(Hour, i64)>,        // cEV by hour
    pub best_hand: Option<HandSummary>,
    pub worst_hand: Option<HandSummary>,
    pub biggest_win: i64,
    pub biggest_loss: i64,
}
```

### Queries to Implement (Rust / session_read_model)

**Load Session Summary** (new)
```rust
pub fn load_session_summary(
```

- Aggregate hands for date + player

- Find best/worst hands
